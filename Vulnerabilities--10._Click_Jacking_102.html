10. Click Jacking


Clickjacking is a type of attack where an attacker tricks a user into clicking on a hidden or disguised element on a web page, often by overlaying it with another transparent or opaque element. The goal of the attack is to trick the user into performing an action that they did not intend to perform, such as making a purchase or changing their account settings.

In web application security, Clickjacking can be used to steal sensitive information or perform unauthorized actions on behalf of the user. For example, an attacker could create a fake login form that overlays the legitimate login form on a website. When the user enters their credentials into the fake form, they are actually submitting them to the attacker's server.

Improper implementation:

// This code loads a malicious iframe that overlays the legitimate login form
var iframe = document.createElement('iframe');
iframe.src = 'https://malicious-website.com/login';
document.body.appendChild(iframe);

Proper implementation:

// This code sets the X-Frame-Options header to prevent the website from being loaded within iframes or other embedded elements
response.setHeader('X-Frame-Options', 'DENY');

In addition to setting the X-Frame-Options header, developers can also use frame-busting code to prevent their websites from being loaded within iframes or other embedded elements. Here's an example of frame-busting code in JavaScript:

if (top != self) {
  top.location.href = self.location.href;
}

This code checks whether the current window is the top-level window, and if not, it redirects the user to the same URL in the top-level window. This effectively breaks out of any iframes or other embedded elements that may be used for Clickjacking attacks.
