Layer 6 - Presentation Layer


• makes the data presentable for applications to consume

• The presentation layer is responsible for translation, encryption, and compression of data

• responsible for translating incoming data into a syntax that the application layer of the receiving device can understand (UTF8 -> ASCII or ASCII -> EBCDIC)

• responsible for adding the encryption on the sender’s end as well as decoding the encryption on the receiver's end so that it can present the application layer with unencrypted, readable data (usually through SSL/TLS)

• compressing data it receives from the application layer before delivering it to layer 5

◇ data compression may be of two types: lossy (data integrity isn't guaranteed) or lossless (data integrity is guaranteed)
